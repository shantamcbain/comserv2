SERVICE DETAILS
================================================================================

DATABASE (MySQL 8.0)
  - Image: mysql:8.0-debian
  - Port: 3306 (internal, exposed in dev only)
  - Storage: mysql_data volume (persistent)
  - Config: config/mysql.cnf
  - Health checks: Enabled
  - Security: Dropped capabilities, no new privileges

CACHE (Redis 7)
  - Image: redis:7-alpine
  - Port: 6379 (internal, exposed in dev only)
  - Storage: redis_data volume (persistent)
  - Memory: 512MB max with LRU eviction
  - Auth: Required password
  - Health checks: Enabled

APPLICATION (Comserv)
  - Image: Built from Comserv/Dockerfile
  - Port: 3000 (behind Nginx in production)
  - Language: Perl 5.40.0
  - Framework: Catalyst
  - Server: Starman (4 workers)
  - User: Non-root (comserv)
  - Health checks: Enabled
  - Dependencies: All CPAN modules pre-installed
  
NGINX (Optional Reverse Proxy)
  - Image: nginx:alpine
  - Ports: 80 (HTTP), 443 (HTTPS with SSL)
  - Features:
    • Static asset caching (30 days)
    • Rate limiting (login: 5/min, API: 10/s)
    • Security headers (X-Frame-Options, CSP, etc.)
    • Gzip compression
    • SSL/TLS ready (commented, enable for production)

PRODUCTION DEPLOYMENT
================================================================================

CHECKLIST
  [ ] Edit .env.production with secure credentials
  [ ] Test database backup/restore
  [ ] Obtain SSL/TLS certificates
  [ ] Configure firewall (ports 80, 443 only)
  [ ] Setup database replication (if needed)
  [ ] Configure monitoring and logging
  [ ] Test health checks
  [ ] Test disaster recovery plan

DEPLOYMENT STEPS
  1. Copy production config: cp .env.production .env.prod.secret
  2. Edit with secure values: nano .env.prod.secret
  3. Build image: docker-compose build
  4. Deploy: docker-compose --env-file .env.prod.secret up -d
  5. Configure SSL: Copy certificates to config/ssl/
  6. Enable SSL in nginx.conf (uncomment section)
  7. Reload Nginx: docker-compose exec nginx nginx -s reload
  8. Verify: curl https://your-domain.com/

TROUBLESHOOTING
================================================================================

SERVICES WON'T START
  docker-compose logs                      # Check error messages
  docker-compose logs database             # Database issues
  docker-compose logs web                  # Application issues
  sudo lsof -i :3000                       # Check for port conflicts
  docker-compose down -v && docker-compose up -d  # Clean restart

DATABASE CONNECTION FAILS
  # Test connectivity
  docker-compose exec database mysql -u comserv -p comserv
  
  # Check database logs
  docker-compose logs database

MEMORY/RESOURCE ISSUES
  docker stats                             # Monitor real-time usage
  docker-compose config | grep resources   # Check limits

NGINX REVERSE PROXY ISSUES
  docker-compose exec nginx nginx -t       # Test config
  docker-compose logs nginx                # Check errors
  docker-compose exec nginx nginx -s reload  # Reload config
