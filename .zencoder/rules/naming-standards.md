# Naming Standards

Use PascalCase for all new file names, function names, and variables (e.g., FileName, GetProjectData). When editing existing code, refactor to PascalCase only if it improves clarity or consistency, and do so minimally to avoid resource-intensive changes. Note style conflicts like PascalCase vs. snake_case, and prefer PascalCase for new elements unless matching existing code.
