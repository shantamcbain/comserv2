Remove login requirement from Proxmox functionality

This commit removes the unnecessary login requirement from the Proxmox
functionality, allowing direct access to Proxmox VM management features
without requiring user authentication. The Proxmox integration now uses
API token authentication exclusively.

Changes:
- Removed admin role checks from all Proxmox controller methods
- Removed login conditional checks from Proxmox templates
- Improved token handling and deobfuscation in ProxmoxCredentials
- Added better error handling and logging for authentication issues
- Created comprehensive documentation of the changes

The application now properly separates API authentication (handled via
tokens) from user authentication, providing a more streamlined user
experience while maintaining security through API tokens.

Documentation:
- Added proxmox_authentication_changes.md with detailed explanation
- Updated proxmox_api.md with new authentication approach
- Created changelog entry in 2024-07-proxmox-authentication-changes.md
